# LivePilot V2 Master Spec

This folder contains the working V2 strategy package for LivePilot.

It combines:

- the high-level product and research critique
- the technical implementation spec
- the prioritized execution plan

The intent is to keep V2 grounded in three questions:

1. what should LivePilot become
2. how should it be built
3. what should be done first

## Intended Use

Use this folder for:

- product direction decisions
- implementation planning
- backlog shaping
- review of scope and priorities

Do not use it as a marketing surface or a tool-count checklist.

The central V2 goal is depth:

- deeper hearing
- stronger retrieval
- better critique
- better co-creation
