The **atlas** gives the AI knowledge of every device in Ableton's library вҖ”
real names, real URIs, real parameters.

The **analyzer** gives it ears вҖ” spectral data from the master bus
via a Max for Live device.

The **memory** gives it history вҖ” a searchable library of production decisions
that persists across sessions.

All three feed into 178 deterministic tools that execute on Ableton's main thread.

### Intelligence

<details>
<summary><strong>Theory вҖ” 7 tools</strong></summary>

<br>

Krumhansl-Schmuckler key detection with 7 mode profiles:
major, minor, dorian, phrygian, lydian, mixolydian, locrian.

Roman numeral analysis via scale-degree chord matching
on a 1/32 note quantization grid.

Voice leading checks вҖ” parallel fifths, parallel octaves,
voice crossing, unresolved dominants.

Species counterpoint generation (1st and 2nd species).
SATB harmonization with smooth voice leading.
Diatonic transposition that preserves scale relationships.

```
analyze_harmony         suggest_next_chord      detect_theory_issues
identify_scale          harmonize_melody         generate_countermelody
transpose_smart
```

</details>

<details>
<summary><strong>Harmony вҖ” 4 tools</strong></summary>

<br>

Neo-Riemannian PRL transforms on the Tonnetz.

```
P  flips the third в”Җв”Җв”Җв”Җв”Җв”Җв”Җ Cm вҶ” C
L  shifts by semitone в”Җв”Җв”Җв”Җ C  вҶ” Em
R  shifts by whole tone в”Җв”Җ C  вҶ” Am
```

All three are involutions вҖ” apply twice, return to origin.

BFS through PRL space finds the shortest voice-leading path
between any two triads. Cm to E major? That's PLP вҖ” the hexatonic pole.
Three steps, each moving one voice by a semitone.
The Hitchcock chord change.

Chromatic mediants for film-score harmony: chords a major/minor third away
sharing 0-1 common tones. Maximum color shift, minimal voice movement.

```
navigate_tonnetz        find_voice_leading_path
classify_progression    suggest_chromatic_mediants
```

</details>

<details>
<summary><strong>Generative вҖ” 5 tools</strong></summary>

<br>

**Euclidean Rhythm** вҖ” Bjorklund distributes N pulses across M steps.
Bresenham's line algorithm applied to rhythm.

```
E(3,8)  = tresillo          Г—В·В·Г—В·В·Г—В·
E(5,8)  = cinquillo          Г—В·Г—Г—В·Г—Г—В·
E(7,16) = Brazilian necklace Г—В·Г—В·Г—Г—Г—В·Г—В·Г—В·Г—Г—Г—В·
```

Layer multiple patterns at different pitches for polyrhythmic textures.

**Tintinnabuli** (Arvo PГӨrt) вҖ” for each melody note, find the nearest tone
of a specified triad. Two voices, one rule, infinite music.

**Phase Shifting** (Steve Reich) вҖ” identical voices with accumulating timing drift.
They start in unison, gradually separate, and eventually realign.

**Additive Process** (Philip Glass) вҖ” melody unfolds note by note.
The structure *is* the composition.

```
generate_euclidean_rhythm    layer_euclidean_rhythms
generate_tintinnabuli        generate_phase_shift
generate_additive_process
```

</details>

<summary><strong>Automation вҖ” 8 tools</strong></summary>

<br>

16 curve types in 4 categories:

```
BASIC в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ linear В· exponential В· logarithmic В· s_curve

SHAPE в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ bezier В· easing
                   (bounce, elastic, back, quad, cubic,
                    quart, quint, expo)

GENERATIVE в”Җв”Җв”Җв”Җв”Җв”Җв”Җ euclidean В· stochastic
```

<details>
<summary><strong>Memory вҖ” 8 tools</strong></summary>

Persistent technique library across sessions.

- **Payload** вҖ” raw MIDI, device params, tempo, URIs

Recall by text query matching mood, genre, texture вҖ” not just names.

claude plugin add github:dreamrec/LivePilot/plugin

| Command | What |
|---------|------|
| `/session` | Full session overview with diagnostics |
| `/beat` | Guided beat creation |

| `/sounddesign` | Sound design workflow |
